1. 프로젝트 한 줄 요약
------------------------------------------------------------
뉴스 기사를 수집해서 명사 간 공동출현 관계를 그래프로 만들고,
노드를 클릭하면 연결 이벤트·가격 수익률·AI 설명이 나오는 웹앱.
v2에서 인과 추출(Gemini), 멀티홉 체인 탐색, 노이즈 대폭 제거가 추가됨.

4. 데이터 입력 3가지 방법

[방법 2] URLs.txt 수동 지정

  python crawler.py --urls
  → 줄 앞에 # 붙이면 주석, 중복 URL 자동 건너뜀
  → 지원 사이트: 연합뉴스, 한경, 매경, 이데일리, 머니투데이, 헤럴드경제,
                 조선비즈, 이인포맥스, 더일렉, 아이로봇뉴스, 지크서,
                 뉴스프라임, 중앙일보, 동아일보, 네이버뉴스(제한적)

[방법 3] news/ 폴더 txt 파일 (네이버 등 봇 차단 사이트 대응)
  backend/news/파일명.txt 로 저장, 형식:
    제목: 기사 제목
    날짜: 2026-03-10   ← 생략 시 오늘 날짜

    본문 내용 (자유롭게)

  → _로 시작하는 파일은 샘플로 인식, 처리 안 됨


5. 파이프라인 흐름 (5단계)
------------------------------------------------------------
[1단계] 크롤링 → crawler.py
  수집 → HTML에서 노이즈 요소 제거(_strip_noise_elements) →
  본문/제목 추출 → DB articles 테이블 저장 (URL 기준 중복 제거)

[2단계] 명사 추출 → noun_extractor.py
  kiwipiepy NNP/NNG/SL 품사 필터 →
  STOPWORDS 제거 (130개+, UI쓰레기/방향어/감정어 포함) →
  FINANCE_KEYWORDS 무조건 유지 →
  Gemini 배치 노이즈 필터 (API키 있을 때만) →
  article_nouns 테이블 저장

[3단계] 그래프 빌드 → graph_builder.py
  공동출현 스코어: 같은 문장+3점, 같은 기사+2점, 같은 날+1점 →
  노이즈 제거: 등장 기사 수 하위1% 제거 + 상위60%(IDF) 제거 →
  하위 20% 엣지 제거 →
  nodes/edges 테이블 저장 + NetworkX 중심성 계산

[3.5단계] 인과 추출 → causal_extractor.py (Gemini 필요)
  기사 본문을 Gemini에 보내 "A → B" 방향성 관계 추출 →
  causal_edges 테이블 저장 (쌓일수록 체인이 정교해짐)

[4단계] 가격 결합 → price_engine.py
  ASSET_NODES(18개 자산) yfinance/pykrx 가격 수집 →
  언급 급증(7일 이동평균 3배) 감지 →
  +1일/+7일/+30일 실제 수익률 계산


6. DB 스키마
------------------------------------------------------------
articles        — 기사 원문 (url UNIQUE)
article_nouns   — 기사별 추출 명사 (position: title/body)
nodes           — 그래프 노드 (noun UNIQUE, total_count, last_seen)
edges           — 공동출현 엣지 (score, article_count, UNIQUE node_a+node_b)
causal_edges    — 인과 엣지 (cause, effect, relation, strength, article_id)  ← v2 신규
price_data      — 자산 가격 히스토리 (ticker+date UNIQUE)
mention_spikes  — 언급 급증 이벤트 + 수익률 (noun+spike_date UNIQUE)

9. crawler.py 핵심 구조
------------------------------------------------------------
DOMAIN_SELECTORS (15개 사이트 전용 셀렉터):
  key: 도메인 문자열 (예: "hankyung.com")
  value: {source, title: [셀렉터 우선순위 리스트], body: [...], extra_headers: {...}}

crawl_single_url(url):
  1) 도메인 매칭 → 전용 셀렉터
  2) 폴백: OG태그 → <title> → 범용 셀렉터
  3) _fetch_with_retry: 최대 3회, 지수 백오프
  4) _strip_noise_elements: 크롤링 직후 UI 쓰레기 제거

_strip_noise_elements(soup):
  script/style/nav/header/footer/aside/iframe 태그 제거
  share, social, sns, kakao, subscribe, reporter, copyright 등
  클래스/ID 키워드로 추가 제거 (네이버/연합/AtozBoard 전용 포함)

AtozBoard CMS 계열 (einfomax, thelec, irobotnews, ziksir, newsprime):
  title: "h3.heading"
  body:  "div#article-view-content-div"
  → 공통 구조라 5개 사이트가 동일한 셀렉터 사용


10. 노이즈 제거 3중 레이어
------------------------------------------------------------
[레이어 1] 크롤러 - _strip_noise_elements()
  HTML 파싱 직후 SNS버튼, 구독버튼, 기자정보, 저작권 등 DOM 요소 제거
  → "okjebo", "카카오톡", "구독", "이미지" 같은 UI 단어가 본문에 포함 안 됨

[레이어 2] 명사 추출 - STOPWORDS (130개+)
  STOPWORDS 주요 범주:
  - 기본 불용어: 것, 수, 등...
  - 뉴스 보도: 기자, 제보, 저작권, 무단전재...
  - 포털 UI: 구독, 이미지, 확대, 카카오톡, SNS...
  - 지명(광범위): 서울, 경기, 뉴욕, 워싱턴...
  - 일반 동작: 확대, 축소, 증가, 강화, 돌파, 달성...
  - 가격 방향: 상승, 하락, 급등, 급락...
  - 규모 수식어: 역대, 사상최고, 최저...
  - 감정/평가: 우려, 기대, 호재, 악재...
  - 매매: 매수, 매도, 거래량...

[레이어 3] 그래프 빌드 - IDF 필터
  전체 기사 60% 이상 등장 노드 자동 제거
  → STOPWORDS를 피해서 들어온 단어도 여기서 차단
  graph_builder.py: MAX_IDF_RATIO = 0.60 (조정 가능)


11. 인과 추출 시스템 (v2 신규, causal_extractor.py)
------------------------------------------------------------
공동출현 그래프의 한계:
  - 방향이 없다 (A↔B, A가 B를 유발하는지 B가 A를 유발하는지 모름)
  - 체인을 못 따라간다 (A-B, B-C가 있어도 A→B→C를 자동으로 보여주지 않음)
  - 모든 기사에 나오는 단어끼리 연결됨 (IDF 필터로 일부 해결)

인과 추출 작동 방식:
  extract_causal_from_article(article_id, title, body):
    → Gemini에 기사 본문 전송
    → "A가 B를 유발/영향/결정" 관계 최대 5개 추출
    → cause, effect, relation(15자), strength(1~3) 반환
    → causal_edges 테이블에 저장

  process_articles_for_causality(batch_size=30):
    → 아직 처리 안 된 기사 배치로 처리
    → causal_edges.article_id 없는 기사 기준

체인 탐색:
  get_causal_chain(noun, depth=3, direction="both"):
    → direction="forward": 이 노드가 유발하는 것들
    → direction="backward": 이 노드를 유발하는 것들
    → direction="both": 양방향 모두

  get_causal_path(noun_a, noun_b):
    → BFS로 A→B 인과 경로 탐색
    → 관계 설명(relation)과 강도(strength) 포함

주의: 기사가 최소 수십~수백 건 쌓여야 의미 있는 인과 체인 형성됨
      처음엔 causal_edges가 없으므로 "인과 데이터 없음" 메시지 정상

  - noun_extractor.py: "gemini-2.0-flash" (노이즈 필터)
  - causal_extractor.py: "gemini-2.0-flash" (인과 추출)
  - api.py (explain): "gemini-2.5-flash" (관계 설명)
  API 키 만료 시: {'code': 400, 'status': 'INVALID_ARGUMENT'} 에러 발생
  → aistudio.google.com/apikey 에서 갱신

[인과 추출 초기 상태]
  처음 실행 시 causal_edges 테이블이 비어있음
  → 프론트엔드 인과 탭: "인과 데이터 없음 — /api/pipeline/extract-causality 실행 필요"
  → POST /api/pipeline/extract-causality 실행하면 기사별 추출 시작
  → 기사 30건당 약 30~60초 소요 (Gemini API 호출)

[IDF 상한 60%]
  데이터가 너무 적을 때 (기사 10건 미만) 오히려 중요 단어가 제거될 수 있음
  → 시드 데이터 27건 기준으로는 일부 단어가 제거됨
  → graph_builder.py MAX_IDF_RATIO = 0.60 → 0.80으로 높이면 완화

[DB 재구축]
  명사 추출 기준(STOPWORDS 등)을 변경했을 때 기존 article_nouns 데이터는
  구버전 기준으로 추출된 것이므로 재처리 필요:
    DELETE FROM article_nouns;
    DELETE FROM nodes;
    DELETE FROM edges;
    → POST /api/pipeline/extract
    → POST /api/pipeline/build-graph

[크롤링 속도]
  crawler.py: 기사 간 0.2~0.5초 대기 (서버 부하 방지)
  RSS 18개 × 80건 + BS 4개 × 40건 = 최대 약 1600건 / 1회 실행
  실제로는 중복 제거 후 수백 건 신규 수집됨
  crawler.log 에 타임스탬프별 성공/실패 기록

  예: "삼성전자" / "삼성" / "삼성전자(005930)" → 하나로 통합
  방향: FINANCE_KEYWORDS와 매핑 테이블 추가

[B] 미국 주식 크롤링
  Reuters, CNBC, MarketWatch는 접근 가능
  Bloomberg, WSJ는 페이월로 불가
  crawler.py DOMAIN_SELECTORS에 영문 사이트 셀렉터 추가 필요:
    "reuters.com": {"title": "h1[data-testid='Heading']", "body": "div.article-body__content"}
    "cnbc.com": {"title": "h1.ArticleHeader-headline", "body": "div.ArticleBody-articleBody"}
    "marketwatch.com": {"title": "h1.article__headline", "body": "div.article__body"}

[C] TF-IDF / Burst 기반 중요도 스코어
  nodes 테이블에 importance 컬럼 추가
  importance = burst_score × IDF
  burst_score: 최근 N일 등장 / 전체 평균 등장 비율
  프론트엔드 노드 크기를 count 대신 importance로 전환
  → "저빈도-고임팩트" 이벤트 노드(후티 같은)가 상위에 노출됨

[D] 인과 그래프 시각화
  현재: 패널 텍스트 목록으로만 표시
  개선: 별도 방향성 D3 그래프 (화살표 있는 DAG)
  인과 엣지를 공동출현 그래프와 레이어로 오버레이

[E] 정기 운영
  매일 1회 전체 파이프라인 자동 실행:
    POST /api/pipeline/full
    POST /api/pipeline/extract-causality
  장 마감 후(21:00~) 실행 권장
